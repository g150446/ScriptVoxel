# プロジェクト概要

## ScriptVoxel - Godot Voxelモジュール デモスイート

### はじめに

ScriptVoxelは、Godot Voxelモジュールの包括的なデモおよびテストスイートで、様々なボクセル地形とゲームプレイシステムを紹介しています。このプロジェクトは、GDScript実装を通じてC++ボクセルモジュールの機能を実証し、Godotでボクセルベースのゲームを開発する開発者に実践的な例を提供します。

### 技術スタック

- **エンジン:** Godot 4.4
- **物理エンジン:** Jolt Physics
- **コア依存関係:** [godot_voxel](https://github.com/Zylann/godot_voxel) C++モジュール
- **スクリプティング:** GDScript

### プロジェクトの目的

このプロジェクトは複数の目的を果たします:

1. **デモンストレーション** - 異なるボクセルレンダリング技術(ブロック状、スムーズ地形)を紹介
2. **テスティング** - 様々なユースケースでgodot_voxelモジュールの機能を検証
3. **教育** - ボクセルゲーム開発を学ぶ開発者に実用的な例を提供
4. **リファレンス** - ボクセルモジュールAPIの実践的なドキュメントとして機能

### 主な機能

#### ブロックボクセルシステム
- Minecraftライクなブロックベース地形
- 完全なブロック編集(配置、削除、ピック)
- ローテーションシステムサポート(軸、Y回転、カスタム)
- メタデータ付きブロックレジストリ

#### 地形生成
- ノイズベースのプロシージャル生成
- マルチパス生成システム
- 構造物の配置(木、植物)
- 水域とバイオーム

#### マルチプレイヤー
- サーバー権限地形同期
- クライアント権限プレイヤー物理
- RPCベースのブロック編集
- UPNPポート転送サポート

#### ボクセルシミュレーション
- ランダムティックシステム(草の広がり)
- キューベースの水シミュレーション
- セルオートマトンパターン

#### スムーズ地形
- Transvoxelアルゴリズムベースのレンダリング
- SDF(符号付き距離場)スカルプティング
- 複雑な形状のメッシュスタンピング
- 可変LOD(詳細レベル)

#### 追加システム
- ボクセルグリッド上のA*パスファインディング
- 流体シミュレーション
- プレイヤーインベントリシステム
- アイテムデータベース
- デバッグ可視化ツール

### メインデモシーン

| シーン | 説明 | 主な機能 |
|-------|------|---------|
| `blocky_game/main.tscn` | フルMinecraftライクゲーム | マルチプレイヤー、インベントリ、地形編集 |
| `blocky_terrain/main.tscn` | シンプルなブロック地形 | 基本的なブロック配置デモ |
| `smooth_terrain/main.tscn` | スムーズ地形デモ | SDFスカルプティング、Transvoxelメッシング |
| `grid_pathfinding/test_grid_pathfinding.tscn` | A*パスファインディング | ボクセル対応ナビゲーション |
| `blocky_fluid/main.tscn` | 流体シミュレーション | 水の流れダイナミクス |
| `multipass_generator/multipass_terrain.tscn` | マルチパス生成 | 複雑な構造物配置 |

### システム要件

**重要な要件:** godot_voxel C++モジュールがGodotエンジンビルドにコンパイルされている必要があります。このプロジェクトはバニラのGodotでは実行できません。

**開発環境:**
- コンパイルされたgodot_voxelモジュールを含むGodot 4.4
- Jolt Physicsサポート(プロジェクト設定で構成済み)

```
│   │   └── gui/                      # ユーザーインターフェース

│   ├── grid_pathfinding/             # A*パスファインディングデモ
│   ├── blocky_fluid/                 # 流体シミュレーション

│   └── addons/                       # プラグインとツール

```

### アーキテクチャ哲学

**C++モジュール統合:** すべてのコアボクセル機能はC++ godot_voxelモジュールに存在します。このプロジェクトのGDScriptコードは以下に焦点を当てています:
- C++ノードタイプの構成
- ゲームプレイロジックの実装
- ユーザーインターフェースとインタラクション
- ネットワーキングと同期

**デモ中心設計:** 各デモは主に自己完結型で、過度な相互依存なしに特定のボクセルモジュール機能を実証します。

**教育的コード:** コードは本番最適化よりも明確さと実証価値を優先します。

### ユースケース

このプロジェクトは以下に最適です:
- Godotでのボクセルゲーム開発の学習
- godot_voxelモジュール機能のテスト
- ボクセルベースのゲームメカニクスのプロトタイピング
- マルチプレイヤーボクセル同期の理解
- プロシージャル地形生成の探索
